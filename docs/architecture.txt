┌─────────────────────────────────────────────────────────────────────────────┐
│                        CONTEXT BUDGET MANAGER                               │
│                                                                             │
│  Each agent declares max_tokens budget before execution                     │
│  check_budget() called before every LLM input                               │
│  Overflow → policy_violation logged (NOT silent truncation)                 │
│  Compression agent triggered if budget exceeded:                            │
│    - Lossless: tool outputs, scores, citations                              │
│    - Lossy: conversational filler only                                      │
│                                                                             │
│  Agent Budgets:                                                             │
│  orchestrator:      8,000 tokens                                            │
│  decomposition:     4,000 tokens                                            │
│  retrieval:         6,000 tokens                                            │
│  critique:          4,000 tokens                                            │
│  synthesis:         6,000 tokens                                            │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│                      EVALUATION PIPELINE                                    │
│                                                                             │
│  15 Test Cases:                                                             │
│  ├── 5 Baseline    (known answers, verifiable)                              │
│  ├── 5 Ambiguous   (underspecified, tests decomposition)                    │
│  └── 5 Adversarial (prompt injections, wrong premises, contradiction traps) │
│                                                                             │
│  6 Scoring Dimensions (all custom, no black-box frameworks):                │
│  ├── answer_correctness      keyword + adversarial resistance check         │
│  ├── citation_accuracy       provenance map vs retrieved chunk IDs          │
│  ├── contradiction_resolution flagged spans resolved in final answer        │
│  ├── tool_efficiency         call count vs max_allowed, expected tools used │
│  ├── budget_compliance       violations count → score penalty               │
│  └── critique_agreement      claim acceptance rate × avg confidence         │
│                                                                             │
│  Each dimension: numeric score [0.0-1.0] + written justification string     │
│  Full reproducibility: exact prompts, tool calls, outputs stored in DB      │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│                    SELF-IMPROVING PROMPT LOOP                               │
│                                                                             │
│  After eval run:                                                            │
│  1. Meta-agent reads all failure cases                                      │
│  2. Identifies worst-performing dimension across failures                   │
│  3. Maps dimension → responsible agent                                      │
│  4. Proposes rewritten prompt with unified diff + justification             │
│  5. Stores proposed rewrite (NEVER auto-applies)                            │
│  6. Human reviews: POST /eval/rewrites/{id}/review                          │
│  7. If approved: re-runs eval on ONLY previously failed cases               │
│  8. Logs performance delta (before/after avg score)                         │
│  All steps timestamped and queryable                                        │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│                    OBSERVABILITY & LOGGING                                  │

│                                                                             │
│  Event types: job_start, agent_start, agent_end, tool_call, tool_result,    │
